- (constraint) Deploy the 4 services (employee-data, auth, firestore-collaboration, pubsub-analytics) as separate microservices via Google Cloud Build
- (decision) Create a shared maven submodule (ems-core) for common entities, DTOs, configurations, and exception handlers, while compiling 4 separate executable Spring Boot services (ems-auth, ems-employee, ems-firestore, ems-pubsub)
- (decision) Cloud Build will build and compile the parent pom.xml, run Docker builds for each of the 4 microservice submodules, push their respective images to Artifact Registry, and run a deployment script to update the running services on the VM
- (constraint) System consists of 4 separate repositories: employee-service-backend (port 8080), department-service (port 8081), employee-service (Angular UI on port 3000), and react-analytics (React UI on port 3001)
